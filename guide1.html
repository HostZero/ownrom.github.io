Build Guide

May 22, 2015

So, here are the steps if you've set up your PC for rom building.
If you haven't, there's a great guide for it on xda by jackeagle, can be found here
Open a terminal, cd to the path of your choice, make a directory for holding the source
Then copy paste the following code into the terminal
repo init -u https://github.com/OwnROM/android.git -b cm-12.1
time repo sync -f
The above step will take quite a while, as it downloads the sources, >=16 GigaBytes of code
If I have your github username, check https://github.com/OwnROM-Devices, and accept the invite
Now follow these instructions, don't fork your device tree
Click on new repository, name should be android_device_vendor-name_device-name, for example for Galaxy S5 international unified, it should be android_device_samsung_klte
Click create, and at the next step, choose import code
Give the link to cm-12.1 device tree for your device, maybe from CyanogenMod, maybe from your device's developers git
Once that's done, proceed to repository, go to the settings, and under default branch, select cm-12.1
Go back to the repo, and under branches, delete all others
Now, click on the file cm.mk, and click on the pencil to edit
There'll be one, or two lines if the device has NFC, which has "vendor/cm"
Change them to vendor/ownrom
There'll also be a line about cm_device-name
Make it ownrom_device-name
Also rename the file to ownrom.mk, you can find that at the top near where you clicked on the pencil to edit.
Then scroll down, and commit changes
Time to edit cm.dependencies, starting with its rename to ownrom.dependencies
Then, where it says repository, add the name of the git where you got it from, if cm, then make it "repository": "CyanogenMod/.......",
Before the first closing }, add the , to the line above, and write
"remote": "github",
"revision": "cm-12.1"

After that should come the closing }
If your device has a common device tree, and that too has dependencies, then copy them into this dependencies file, and edit as explained
Make sure, that within a pair of { }, all lines except the last end with a , and that all { } except the last set, end with a ,
Since I'm sure you've got it by now, add an entry to vendor blobs for your device here, generally they can be found at TheMuppets GitHub
Refer to these commits for help
https://github.com/OwnROM-Devices/android_device_moto_shamu/commit/b42aa2774d368c5aa186021d526a822bc5547ef5
https://github.com/OwnROM-Devices/android_device_moto_shamu/commit/6ef60345d598c466282ac0788d3e44854ce7437b
https://github.com/OwnROM-Devices/android_device_moto_shamu/commit/c3c113a350bb1e465f83e0b80c4e69274302fc3d
Lastly, vendorsetup.sh
Whatever the contents are, delete it, and make it
add_lunch_combo ownrom-device-name_userdebug
Commit it
And that's it, you've done ownifying!!!!
Now once the repo has finished syncing with no errors, go back to the terminal
. build/envsetup.sh
time brunch device-name
The brunch command, will fetch device tree and dependencies of your device, and build ownrom for it :D

Best of luck, bye!!!
